1. Что это?
xbData - это слой абcтракции для работы с простыми моделями данных.
Собственно входными данными для модели служит построенный особым
образом массив.

$data = array(
  'table'  => 'имя_таблицы_в_бд_без_префикса',
  'fields' => array(
  // Поля в специальном формате
  ),
  'add' => array(
    'table' => 'имя_таблицы_значений_допполей_в_бд_без_префикса',
    'field' => 'имя_столбца_связанного_ресурса_в_таблице_допполлей'
  ),
  'indexes' => массив_индексов,
  'unique'  => массив_уникальных_ключей
);

Формат полей - особым образом сформированный массив.
  id           - идентификатор поля в базе полей (0)
  type         - полная информация по типу поля (0):
    0x00000007 : первичный тип
             0 - логическое значение
             1 - целое число
             2 - строка, текст
             3 - число с плавающей запятой
             4 - набор
             5 - структура
             6 - дата, время
             7 - переменная (служебный тип)
    0x00000008 - не null (not null)
    0x000000f0 : тип ввода, субтип (зависит от первичного типа)
    0x00ffff00 : обработчики значения
    0x40000000 - первичный ключ
    0x20000000 - автоинкремент
    0x10000000 - unsigned, binary
  flags        : флаги поля (null) :
    0x00000001 - обязательное поле
    0x00000002 - повторение в форме
    0x00000004 - скрытое поле
  length       - длина поля (null)
  default      - значение по умолчанию (null)
  elements     - сериализованный массив возможных значений (null)
  input        - элемент ввода (null)
  regexp       - регулярное выражение для проверки или замены (null)
  replace      - замена (null)
  strip        - регулярное выражение для удаления лишнего (null)
  access       - доступ к полю (null)
  external     - внешнее поле (не поддерживается)
  key          - ключ


Входные данные должны быть переданы в конструктор класса xbDataModel.
Внутри конструктора эти данные автомтически корректируются. Обратите
пристальное внимание! Для того, чтобы взаимодействовать с кешем,
необходимо корректировать модель из родительского объекта методом
xbDataModel::correct, отдавая конструктору модели уже скешированные
данные (см. пакет xbNode).

Объект xbDataModel содержит два основных метода:

- form  - получение формы для запроса; возвращает массив полей;
- query - формирование запроса; возвращает объект xbDataQuery, снабжённый
          т.н. chain-методами, позволяющими изменять запрос цепочками
          вызовов.

2. Компоненты.
xbData         - библиотека функций
xbDataTypes    - библиотека типов
xbDataHandlers - библиотека обработчиков данных
xbDataFields   - библиотека функций для работы с полями
xbDataModels   - библиотека функций для работы с моделями
xbDataModel    - класс модели данных
xbDataQuery    - класс запроса
